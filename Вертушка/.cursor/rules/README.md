# Правила Cursor AI для проекта Вертушка

Этот каталог содержит правила (rules) для Cursor AI, которые помогают поддерживать консистентность кода и архитектуры проекта.

## Структура правил

- **`vinyl-records.mdc`** — правила работы с данными виниловых пластинок
- **`api-conventions.mdc`** — конвенции для API endpoints (всегда применяется)
- **`mobile-conventions.mdc`** — конвенции для мобильного приложения
- **`database.mdc`** — правила работы с базой данных
- **`scanning.mdc`** — правила для функций сканирования

## Как использовать правила

### Автоматическое применение

Правила с `alwaysApply: true` применяются автоматически ко всем соответствующим файлам.

Правила с `globs` применяются автоматически при редактировании файлов, соответствующих шаблонам.

### Ручное применение

Используйте `@` символ для явного применения правил:

```
@vinyl-records Добавь поле condition в модель Record
```

```
@api-conventions Создай endpoint для поиска пластинок по артисту
```

### Комбинирование правил

Можно применять несколько правил одновременно:

```
@vinyl-records @database Добавь новую модель для хранения состояния пластинок
```

## Добавление новых правил

1. Создайте новый файл `.mdc` в этой папке
2. Добавьте метаданные в начале файла:
   ```markdown
   ---
   description: Описание правила
   globs:
     - "**/*.py"
   alwaysApply: false
   ---
   ```
3. Напишите правила в формате Markdown
4. Добавьте примеры кода и ссылки на файлы

## Обновление правил

Правила должны обновляться по мере развития проекта. Не забывайте:
- Обновлять примеры кода
- Добавлять новые паттерны
- Удалять устаревшие правила
