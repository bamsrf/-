# PLAN: P2P-маркетплейс Вертушка

> Детальный технический+бизнес спек для M8. Родительский документ — [PLAN_MONETIZATION.md](PLAN_MONETIZATION.md). Здесь только Поток B (P2P) — полная архитектура, юр-обвязка, антифрод, фазный rollout.

## TL;DR

- **Что строим:** C2C-маркетплейс винила. ИП-«Вертушка» работает агентом (ст. 1005 ГК), ЮKassa держит эскроу через продукт «Безопасная сделка», продавцы — самозанятые/физлица, доставка CDEK/Boxberry с PVZ-приёмкой и видео распаковки.
- **Что НЕ делаем сразу:** покупателей-юрлиц, B2C-магазины, личную встречу, домашнюю доставку без видео, веб-аппу, мульти-валюту.
- **Кого копируем:** Авито Доставка (flow + видео распаковки в ПВЗ), Discogs (Goldmine grading, привязка к release_id, dead-wax), Vinted (UX доверия, blind reviews), Reverb (seller protection policy).
- **Финансовая модель:** комиссия 7% с продавца + 1.5% буферный сервис-фи с покупателя; take rate ≈ 8.5% GMV; налог ИП = только с комиссии. Чистая маржа на сделке ~2.5% GMV.
- **Длительность Phase 2 (P2P):** 16–21 человеко-неделя чистого dev времени (sole), растянутые на 7–8 календарных месяцев.

## B.1. Эталонные кейсы — что забираем

| Источник | Что заимствуем |
|---|---|
| **Авито Доставка / Юла** | Flow «оплатил → отправил → ПВЗ принял с видео → 24–72ч на жалобу → release». РФ-стандарт ожидания C2C |
| **Discogs Marketplace** | Goldmine grading, привязка к release_id, dead-wax/матрица как обязательное фото, два независимых рейтинга «record condition» + «sleeve condition» |
| **Vinted** | Простая публикация в 4 шага, blind reviews, калькулятор fee перед публикацией |
| **Reverb** | Trust Badges на основе объективных метрик, понятный seller protection policy |

**Что НЕ копируем:** open feed маркетплейса (без рекламной ленты — нарушает принцип ROADMAP «не реклама»), свободные условия пластинки (только Goldmine), любую доставку (винил требует специальной упаковки), мульти-валюту (Phase 2 — только RUB).

## B.2. Юридическая обвязка (несущая конструкция)

### B.2.1. Агентская схема (ст. 1005 ГК РФ)

ИП = агент, продавец = принципал, покупатель = заказчик услуги. Платёж проходит транзитом через эскроу ЮKassa; нам падает только агентское вознаграждение.

**Что прописать в оферте-агентском договоре с продавцом:**
- Размер вознаграждения 7% (без учёта доставки) + право удержания из платежа покупателя
- Право одностороннего изменения условий с 30-дневным уведомлением
- Право блокировки при нарушении правил
- Отказ от ответственности за качество товара (это товар принципала)
- Обязанность продавца самостоятельно декларировать НДФЛ/НПД
- Согласие на передачу ПДн покупателю в рамках исполнения сделки
- Подсудность по месту регистрации ИП

**Юридические риски:**
- **Переквалификация в куплю-продажу** (если у нас централизованный склад/контроль цены) → НДС+НДФЛ+штрафы. Митигация: нет склада, цену ставит продавец, доставка идёт напрямую от продавца к покупателю.
- **Реклассификация продавцов в работников** → митигация: одинаковые условия для всех, нет «штатных» продавцов.

### B.2.2. 54-ФЗ (онлайн-кассы)

- Касса нужна **только на агентское вознаграждение** (наши 7% + 1.5% сервис-фи), не на GMV
- Тип чека: **«Агентский»** (`agentType=AGENT`), выбивается в момент release
- Cloud-касса через ЮKassa (АТОЛ-Онлайн / Чек-Онлайн), не покупаем физический ФР. Cost ~1700₽/мес + ~20к₽ настройка.

### B.2.3. Налоги ИП

- УСН 6% платится с агентского вознаграждения, не с GMV. Лимит 2026 ~265 млн₽ — некритичен на ближайшие годы.
- Пример: лот 10 000₽ → наша комиссия 850₽ → налог 51₽.

### B.2.4. Самозанятые продавцы

- В личке продавца кнопка «Связать с Мой налог» через ЮKassa → выше KYC-лимит (до 300к/мес vs 30к), плашка «Самозанятый» в карточке.
- ЮKassa автоматически регистрирует чек НПД через API ФНС после release.

### B.2.5. 152-ФЗ и ЗоЗПП

- ИП регистрируется оператором ПДн (Реестр через Госуслуги, бесплатно)
- ЗоЗПП **не применяется** к C2C — в оферте покупателя явно: «Сделка между физлицами, защита — через диспуты платформы»
- Это критично для подачи в Apple/Google — позиционируем физический товар между физлицами, не магазин

### B.2.6. 6 документов на сайте

1. Terms of Use
2. Политика обработки ПДн
3. Оферта-агентский договор продавца
4. Оферта-договор покупателя
5. Правила публикации лотов
6. Правила диспутов

Бюджет на юриста: 30–50к₽.

## B.3. State machine сделки

```
Listing:    Draft ─moderation 24h─> Active ─seller deletes─> Archived
                                       │
                                       └─buyer clicks buy─> Reserved
Order:      Reserved ─payment 15min─> Paid ─track 72h─> Shipped
              │ timeout                │ timeout 72h
              └→ Cancelled             └→ Cancelled + refund + штраф seller
            Shipped ─courier API delivered─> Delivered → AwaitingInspection
            AwaitingInspection (72h) ─silent timeout─> Completed
                                  ├─buyer "Confirm"─> Completed
                                  └─buyer "Open dispute"─> Disputed
            Disputed ─admin 48h─> Refunded | Released | PartialRefund
            Completed → ReviewedByBuyer → ReviewedBySeller → Closed (14 дней окно)
```

**Ключевые временные окна:**
- 15 мин — на оплату после Reserve
- 72ч — продавцу на ввод трека после Paid
- 72ч — покупателю на проверку после Delivered (auto-release при молчании)
- 24ч — продавцу на ответ в диспуте (иначе auto-decision в пользу покупателя)
- 48ч — SLA админ-арбитража
- 14 дней — окно blind-отзывов после Completed
- 1 год — хранение доказательной базы споров

**Refund-окно после Completed:** **нельзя вернуть** — для устранения «buyer's remorse». Только через диспут до Completed.

## B.4. Модели данных (новые таблицы Backend)

| Модель | Назначение |
|---|---|
| `Listing` | Лот: seller_id, record_id (FK Discogs), pressing_variant, record_grade, sleeve_grade, price_rub, status, shipping_methods, moderation_status |
| `ListingPhoto` | Обязательный набор: cover_front, cover_back, record_side_a/b, label, dead_wax. Минимум 6. |
| `ListingDefect` | Декларация дефектов: kind (scratch/warp/scuff/split_seam/ring_wear/water_damage/mold), severity, photo |
| `Order` | Сделка: listing_id, buyer_id, seller_id, prices, commissions, status, shipping_method, tracking_id, временные метки всех переходов |
| `OrderEvent` | **Иммутабельный audit log** всех переходов: event_type, actor_id, payload, ts. Append-only. |
| `Shipment` | carrier, tracking_id, PVZ from/to, weight/dims, insurance, статус-зеркало карриера, events |
| `Dispute` | order_id, opened_by, reason (not_received/condition_mismatch/fake/damaged/wrong_item), description, status, resolution |
| `DisputeEvidence` | uploaded_by, kind (video_unboxing/photo_packaging/photo_defect/chat_screenshot), url |
| `Review` | Blind отзывы: 4 параметра (accuracy, communication, packaging, shipping_speed) + overall + actual_grades если расходятся с заявленными |
| `SellerProfile` | 1:1 с User: kyc_level, tax_status, payout_method, total_sales/volume, rating, response_time, badges, strikes (anti-disintermediation) |
| `PaymentEvent` | Лог ЮKassa: yookassa_payment_id, yookassa_safe_deal_id, event_type, payload (raw). UNIQUE на (payment_id, event_type) для идемпотентности |
| `BannedListingPattern` | Модерация: pattern_type (keyword/image_hash/discogs_id_blacklist), value, reason |

**Расширения существующих:**
- `BlockedContact` → +enum типы `phone`, `device_id`, `card_hash` (хеш от ЮKassa)
- `CollectionItem` → связь с Listing через record_id; после Completed опция «удалить из коллекции»
- `User` → relationship на `seller_profile`

## B.5. Антифрод-стек по слоям

### Слой 0 — регистрация
- Phone binding обязателен перед первой публикацией/покупкой
- Email-верификация strict (без неё нельзя купить)
- Device fingerprinting (Expo Application APIs), хранение последних 5 устройств
- Apple Sign In: relay-email принимаем, но phone binding отдельно

### Слой 1 — KYC продавца (tiered)

| Уровень | Требования | Лимит/мес |
|---|---|---|
| Lite | Email + phone | 30 000₽ |
| Mid | + паспорт + селфи (ЮKassa Identity ~50₽/проверка) | 300 000₽ |
| Full | + видео-верификация + статус самозанятого/ИП | без лимита |

### Слой 2 — модерация листинга
- Auto: regex запрещённых слов, Discogs ID валидация, минимум 6 фото, ML NSFW (OpenAI Vision ~10₽/листинг), OCR названия vs Discogs metadata
- Manual: 100% новых продавцов (<3 сделок), 5% spot-check остальных, 100% при сработке `BannedListingPattern`
- Trust-skip: seller с trust_score >80 и >50 сделок → auto-approve если ML чист
- SLA: 24ч max, алерт если очередь >50

### Слой 3 — антифрод покупателя
- Velocity check: регистрация → покупка <5 мин = challenge (3D Secure)
- Disposable email домены — блок-лист
- IP-репутация (MaxMind / IPQualityScore)
- Лимит 3 активных Reserved на 24ч
- Card hash blacklist по чарджбэкам (от ЮKassa)

### Слой 4 — anti-disintermediation (КРИТИЧНЫЙ для маржи)

Это **самая критичная защита** для C2C — продавцы и покупатели хотят уйти в Telegram чтобы не платить комиссию.

1. **Контакты скрыты до Reserved** (username виден, почта/телефон/адрес нет)
2. **Чат-фильтр** **заменяет** контактные данные на `***скрыто***` (не блок, чтобы пользователь продолжал общаться). Регексы: РФ-телефоны во всех форматах (+7, 8, без префикса), `@username`, t.me/tg://, email, ВК id
3. **Strike-система**: 3 страйка = бан. Видимо в профиле «warnings: N/3»
4. **Эскроу как value-add**: в каждой карточке/чате плашка «Защита 14 дней»
5. **В FAQ явно**: «вне платформы — без защиты»
6. **Honeypot**: «продавец» с плохими лотами тестит сговорчивость покупателя (warning, не блок)

### Слой 5 — момент доставки
- **Видео распаковки в ПВЗ обязательно** для лотов >3000₽ (CDEK/Boxberry имеют услугу «приёмка с видео»)
- **ID-код сделки** (`VRT-A4F2-91X3`) — продавец **обязан** наклеить на упаковке для доказательства тождественности
- Домашняя курьерская доставка (Яндекс/СберКурьер) — только Full KYC + крупные города + обязательное видео распаковки в 24ч
- **Самовывоз/личная встреча — ЗАПРЕЩЕНО** до Phase 2.6

### Слой 6 — disputes
- Окно подачи 72ч после Delivered
- Доказательная база required per reason:
  - `not_received` — скриншот трекинга + переписка с курьером
  - `condition_mismatch` — видео распаковки + крупные фото + ссылка на описание
  - `fake` — фото dead-wax + сравнение с Discogs reference
  - `damaged_in_transit` — видео распаковки в ПВЗ
- 24ч ответа продавца → auto-decision при молчании
- Manual review обязателен для disputes ≥5000₽ или повторяющихся сторон
- Доказательная база хранится 1 год

## B.6. Винильная специфика

### Goldmine grading (обязательно раздельно для пластинки и конверта)
M / NM / VG+ / VG / G / F-P. Dropdown с tooltips и фото-примерами. При `record_grade ≤ G+` — красная плашка «состояние слабое, изучите фото».

### Pressing variant обязателен
Страна + год + лейбл-код + dead-wax матрица. Подсказки из Discogs при выборе release_id (автоматически предлагаются варианты pressing'ов с release).

### Обязательные 6 фото
`cover_front`, `cover_back`, `record_side_a`, `record_side_b`, `label_close`, `dead_wax_close`. Опциональные +3 (внутренний конверт, insert, дефекты) дают bonus в trust_score.

### Defects declaration
Чек-лист с обязательной фотофиксацией для `severity=major`. Скрытие дефектов → автоматический win диспута для покупателя.

### Запрещённый контент
- Бутлеги (флаг через Discogs `Unofficial Release`)
- Контрафакт (community flagging + matrix verification)
- Экстремистский контент на обложке (ML + manual)
- Не-винил под видом винила
- «Конверт без пластинки» (запрещено в Phase 2)

## B.7. Доставка

| Карриер | Использование | Особенности |
|---|---|---|
| **CDEK** | Приоритет 1 | Стандарт для коллекционеров, услуга «осторожно: винил», PVZ-приёмка с видео |
| **Boxberry** | Fallback | Дешевле в средних городах, PVZ-приёмка |
| **Почта России** | Last resort | Алерт покупателю «риски выше» |
| **Яндекс/Сбер курьер** | Премиум | Full KYC + МСК/СПб + лоты >20к |
| **Avito Доставка** | Не используем | API закрыт для конкурентов; рассмотреть в Phase 3+ |

Интеграция: `POST /v2/orders` (регистрация заказа), webhook на `ORDER_STATUS` (received/in_transit/delivered/returned), расчёт стоимости через API с кешированием 24ч по (from_city, to_city, weight). Страховка **обязательна** на лоты >5000₽ (1.5% от стоимости через CDEK).

**Гайд по упаковке для продавца:** картонная коробка LP-mailer + два угла + bubble wrap. В UI продавца чек-лист «упаковали?».

## B.8. Эскроу через ЮKassa «Безопасная сделка»

- Отдельный договор с YooMoney (поверх обычного эквайринга)
- Технически: `payment.create` с `capture=false` + `transfers=[{account_id, amount, platform_fee_amount}]` → webhook `payment.waiting_for_capture` → `payment.capture` → деньги в эскроу до `safe_deal.confirm` / `safe_deal.cancel`
- **Комиссия с 1 января 2026: до 4.5% + НДС 22% от комиссии ≈ 5.5% эффективно** (НДС идёт ЮKassa, не нам)
- Все вебхуки с HMAC подписью — verify обязательно
- Hold до 90 дней

**Альтернативы:** Tinkoff Pay Безопасная сделка (хуже доки), CloudPayments эскроу (5–6%). Рекомендация — **ЮKassa** как самый отработанный путь.

## B.9. Экономика комиссий — пример лота 10 000₽

| Статья | Сумма |
|---|---|
| Цена лота | 10 000₽ |
| Доставка (продавцу→покупателю, оплачивается отдельно) | 500₽ |
| Сервис-фи Вертушки с покупателя (1.5%) | +150₽ |
| **Покупатель платит** | **10 650₽** |
| Комиссия ЮKassa (5.5%) | −586₽ |
| Наша комиссия с продавца (7%) | +700₽ |
| **Продавцу уходит** | 10 000 − 700 = **9 300₽** |
| **Наша gross маржа** | 700 + 150 − 586 = **264₽** |
| УСН 6% | −16₽ |
| **Чистыми на сделке** | **~248₽ (2.5% GMV)** |

**Вывод:** маржа тонкая, для устойчивости нужен либо рост среднего чека (для винила реально >5000₽), либо повышение комиссии до 9–10% (как Discogs) после валидации.

## B.10. UX и trust signals

### Карточка листинга (приоритет сверху вниз)
1. Главное фото + горизонтальный свайп ко всем 6+
2. Артист — Альбом (с привязкой к Discogs)
3. Цена + CTA «Купить с защитой 14 дней»
4. Pressing variant (страна, год, лейбл, матрица)
5. Состояние: пластинка [VG+] | конверт [NM] — раздельные плашки
6. Defects (раскрывающийся блок с фото)
7. Описание + стоимость доставки (калькулятор по выбранному городу)
8. Профиль продавца: аватар, рейтинг, badges, кол-во сделок, дата регистрации, response time, % успешных, KYC статус
9. «Задать вопрос» (открывает чат до покупки)

### Страница сделки (order)
- Progress bar сверху: Reserved → Paid → Shipped → Delivered → Completed
- Время оставшееся до auto-action: «Авто-подтверждение через 18 ч 24 мин»
- Свёрнутая сводка: артикул + фото + цена
- Чат с pinned summary
- CTA в зависимости от state: «Ввести трек» / «Подтвердить получение» / «Открыть спор»
- История событий (можно развернуть)

### Badges (формула прозрачна в FAQ)

| Badge | Условие |
|---|---|
| Verified Seller | KYC mid+ |
| Trusted Seller | 10+ сделок, rating ≥4.7, dispute <3% |
| Vinyl Expert | 50+ сделок винила, rating ≥4.8 |
| Quick Responder | avg response <30 мин за месяц |
| Power Seller | 200+ сделок, rating ≥4.9 |

### Push-уведомления — принцип «только по делу»
**Разрешено:** payment, ship, delivery, dispute, money on the way, review reminder (1 раз D+7).
**Запрещено** (нарушает принципы ROADMAP): «скоро закроется», «не упустите», «кто-то смотрит», daily-ремайндеры.
