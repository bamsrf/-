## Чек-лист подготовки данных перед письмом

Перед отправкой собери **конкретику** через SQL:

```sql
-- 1. Сколько кликов на конкретный магазин за месяц
SELECT count(*) AS clicks
FROM offer_clicks oc
JOIN store_listings sl ON sl.id = oc.listing_id
JOIN stores s ON s.id = sl.store_id
WHERE s.slug = '{shop_slug}'
  AND oc.created_at >= now() - interval '30 days';

-- 2. Топ-10 пластинок которые юзеры кликали
SELECT r.artist, r.title, count(*) AS clicks, sl.url
FROM offer_clicks oc
JOIN store_listings sl ON sl.id = oc.listing_id
JOIN stores s ON s.id = sl.store_id
JOIN records r ON r.id = sl.matched_record_id
WHERE s.slug = '{shop_slug}'
  AND oc.created_at >= now() - interval '30 days'
GROUP BY r.artist, r.title, sl.url
ORDER BY clicks DESC
LIMIT 10;

-- 3. Уникальные юзеры (если хотим аргументировать «не боты»)
SELECT count(DISTINCT coalesce(user_id::text, ip_hash)) AS unique_visitors
FROM offer_clicks oc
JOIN store_listings sl ON sl.id = oc.listing_id
JOIN stores s ON s.id = sl.store_id
WHERE s.slug = '{shop_slug}'
  AND oc.created_at >= now() - interval '30 days';
```

Скриншот в письмо: их Google Analytics → Audience → Traffic Sources → отфильтруй `utm_source=vertushka`. Если они видят те же цифры что и у нас в БД — это доверие.

## Условия — что обычно выходит на договорённости

| Магазин-сегмент | Комиссия | Кд-метод | Юр.форма |
|---|---|---|---|
| **Крупные** (Plastinka.com, Audiomania) | 3-5% | Промокод + UTM-сверка | ИП ↔ ИП через ЭДО |
| **Средние** (Vinyl.ru, Vinylmarkt, Vinylpark) | 5-7% | Промокод | ИП ↔ ИП или самозанятый ↔ ИП |
| **Малые / нишевые** (региональные шопы) | 5-10% | Промокод / прямая ссылка | самозанятый ↔ ИП, чек по факту |

Промокод выгоден всем:
- **Юзеру** — реальная скидка, мотивация кликнуть именно через нас
- **Магазину** — отслеживание трафика без интеграций с CPA-сетями
- **Нам** — недвусмысленный учёт «откуда заказ»

## После договорённости — что заполняешь

Один SQL-апдейт на магазин:

```sql
UPDATE stores SET affiliate_program = '{
  "type": "direct",
  "deeplink_template": null,
  "params": {},
  "commission_pct": 5.0,
  "promo_code": "VERTUSHKA10",
  "contact": "owner@plastinka.com",
  "negotiated_at": "2026-06-15",
  "payout_method": "bank_transfer",
  "notes": "5% от чистого заказа, выплата 1-го числа месяца, чек на ИП Иванов"
}'::jsonb
WHERE slug = '{shop_slug}';
```

После этого `wrap_url()` начнёт добавлять только UTM (без affiliate-обёртки — магазин сам читает UTM в GA, отчитывается раз в месяц).

Для будущих автоматических выплат (Phase C) — поле `commission_pct` будет браться cron-задачей и сравниваться с заказами магазина (которые либо они нам присылают XML/CSV, либо мы парсим из их админки если дадут доступ).
