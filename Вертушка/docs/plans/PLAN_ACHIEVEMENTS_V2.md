# План: Система ачивок Вертушки v2

> Переработка [PLAN_ACHIEVEMENTS.md](PLAN_ACHIEVEMENTS.md) (1115 строк → ~570). Согласовано в обсуждении 2026-05-10/11.
>
> **Что изменилось относительно v1:**
> - Структура: **серии** (видимые с инструкцией) + **рандомные** (полностью скрытые, влётные).
> - Названия: раздел = «Ачивки» (не «Зал трофеев»), группы = «Серии» (не «Созвездия»).
> - Сезонные окна выкинуты: бронь подарков идёт заранее, ломается привязка к датам.
> - Очки/уровни выкинуты: только косметика напрямую от конкретных ачивок.
> - Welcome ceremony выкинут: главный момент — A1, без принудительных серий.
> - Стилистика — **эмалевые пины**, тиры названы по-русски и кодируются цветом из единого градиента (светло-голубой → розовый → тёмный).
> - Ачивки видны не только на публичном web-профиле, а и на внутри-приложенческих профилях (`Mobile/app/user/`).
> - Шеринг ачивки = отдельный экран генерации картинки под Instagram Stories (9:16) с автокропом под 1:1 и 4:5.
> - Импорт Discogs / редактирование метаданных / spinner на детальной — отложено до появления фич.
> - Стартовый каталог: **10 серий + ~12 рандомных = ~65 ачивок** (вместо 190).

## 1. Принципы

| Принцип | Смысл |
|---|---|
| **Глубина > количество** | «100 пластинок» — слабо. «10 разных стран пресса» — сильно. |
| **Видимое и невидимое** | Серии — открытая дорога с инструкцией. Рандомные — скрыты целиком, без счётчиков, без подсказок. |
| **Никаких daily-механик** | Нет streak'ов, нет «открой 7 дней подряд», нет дедлайнов. |
| **Награды — косметика** | Бейджи-пины, рамки, темы, титулы, особые иконки коллекций. Никакой валюты, премиума, скидок. |
| **Анти-grind** | Все массовые ачивки требуют разнообразия (N **разных** стран/жанров), не N штук. |
| **Один раз заработал — навсегда** | Удалил пластинки и упал ниже порога — ачивка остаётся. L-ачивки фиксируют исторический максимум. |
| **Без процентов** | В UI только «открыто X из Y». Никаких процентов и графиков статуса. |

## 2. Архитектура

### 2.1. Серии (achievement series)

- Видимы в каталоге с названием, описанием и инструкцией («собери N стран», «подари N разных людей»).
- 4–8 ачивок внутри, выстроенных в прогрессию (B1 → B2 → … либо параллельные ветки).
- Закрытие серии (все ачивки внутри) = **мета-ачивка** уровня Эпическая/Легенда + уникальная косметическая награда.
- Прогресс по конкретной ачивке внутри серии виден только когда юзер получил хотя бы одну из этой серии (= вступил в дорогу).

### 2.2. Рандомные

- В каталоге одна общая карточка: **«❓ Сюрпризы»** с подзаголовком «Их обычно находят сами».
- Полностью скрыты до анлока: ни названия, ни условия, ни счётчика.
- Анлок — лёгкий всплывающий тост в приложении, без push-сирены. Сохраняем эффект «не может быть».

### 2.3. Когда что использовать

| Тип ачивки | Рандомная | Серия |
|---|---|---|
| Прогрессия по тиру / количеству | — | ✅ B, J |
| Тематическая дорога | — | ✅ D-страны, F-жанры |
| Шутка / синхрония | ✅ «Тёзка», «Тридцать три» | — |
| Зависит от поведения мира | ✅ EE_time_machine | — |
| Тестирует все режимы фичи | — | ✅ INV-приглашения |

### 2.4. Где живут ачивки

Три точки показа:

| Место | Что показываем | Кем доступно |
|---|---|---|
| **Свой профиль** (`Mobile/app/profile.tsx`) | Главный блок «🏆 Ачивки» с превью топ-3 свежих и CTA на полный грид. | Только владелец. |
| **Чужой профиль в приложении** (`Mobile/app/user/[username].tsx`) | Блок «🏆 Ачивки» с топ-3 свежих + общий счётчик «12 ачивок». Тап → полный грид этого юзера. | Любой залогиненный юзер при заходе через поиск/follow. |
| **Публичный web-профиль** (`ProfileShare`) | Компактный блок топ-3 свежих. Без полного грида (страница и так длинная). | Любой по ссылке, без логина. |

**Видимость рандомных:** на любом профиле они показываются как «❓ Получено 7 сюрпризов», без раскрытия названий. Intrigue-эффект для зрителя.

**Видимость L-категории (стоимость коллекции):** только если `ProfileShare.show_collection_value=true`, иначе скрыто и на in-app, и на web.

## 3. Тиры, стилистика, бенчмарки

### 3.1. Тиры

5 тиров, кодируются цветом из единого градиента приложения (светло-голубой → розовый → тёмный). Чем темнее, тем эпичнее и реже.

| Тир | Цвет | Доля юзеров (цель) |
|---|---|---|
| 💧 **Простая** | Светло-голубой `#A5C8E1` | ≥30% |
| 🔵 **Заметная** | Голубой `#5B7DD8` | 5–30% |
| 🌸 **Редкая** | Розовый `#E89AC0` | 1–5% |
| 🌌 **Эпическая** | Тёмно-синий `#1B237D` | 0.1–1% |
| ⚫ **Легенда** | Графит `#0A0A1A` | <0.1% |

После Phase 1 — прогон бэкфилла на реальной базе, калибровка порогов так, чтобы доли совпали. Меняем цифру, не описание.

> **Альтернативный набор имён** (если хочется винил-метафоры): Сингл → EP → LP → Box-сет → Тест-прессинг. Финальное решение — за тобой.

### 3.2. Дизайн пинов

Каждая ачивка — **эмалевый пин**: круг/щит/гексагон со специфическим узором и металлической каймой. Референсы пользователя: UFO welcome, Tyler the Creator IGOR/Flower Boy, 163 spaceship, Mokita pins set.

**Правила художника:**
- Чёткая силуэтная иконка по центру (узнаваемая в 24×24).
- Цветная заливка → тир (см. §3.1).
- Металлическая обводка → 2 пикселя, цвет зависит от тира (Простая = матовое серебро, Заметная = серебро, Редкая = розовое золото, Эпическая = синее золото, Легенда = графит с холо-эффектом).
- Размеры для UI: 64×64 в гриде, 256×256 в анлок-анимации, 700×700 в share-card.
- Формат: SVG для UI, PNG 1024×1024 для share-card-генерации.
- Каждый пин рисуется как **отдельный артефакт** (не процедурно). Каталог = папка `Mobile/assets/achievements/<code>.svg`.

### 3.3. Бенчмарки (как сделано у других)

| Приложение | Что взять |
|---|---|
| **Apple Fitness — Awards** | 3D-визуализация значков, тап → детальная карточка с анимацией. Главный визуальный focal point профиля. |
| **Strava — Achievements** | Сразу готовая share-card на каждый анлок. |
| **Pokémon GO — Medals** | Тиры через цвет рамки самой медали (bronze/silver/gold/platinum) — иерархия видна без подписи. |
| **Untappd — Badges** | Грид квадратных значков с category-фильтрами. |
| **Discord — Profile Badges** | Тонкая полоска значков рядом с ником — компактное превью. |
| **Robinhood — Trophy Room** | Отдельный экран с подсветкой свежих наград, не запихивают в общий профиль. |
| **GitHub — Achievements** | Маленькие пины рядом с аватаром в превью контрибьютора. |

**Что берём для Вертушки:**
- Apple Fitness — focal point своего профиля.
- Strava — каждый анлок генерит share-card.
- Pokémon GO — тиры через цвет (см. §3.1).
- Untappd — грид с фильтрами по сериям.
- Discord — превью топ-3 на чужом профиле.

| `R_new_year` | **Первая в году** | Добавил первую пластинку года с 00:00 до 00:30 1 января. |
| `R_friday_night` | **Пятничный спин** | Добавил пластинку в пятницу с 22:00 до 02:00. |
| `R_leap_day` | **29 февраля** | Добавил пластинку 29 февраля. |

### 5.3. Самореферентные

| Код | Название | Триггер |
|---|---|---|
| `R_self_titled` | **Тёзка** | Пластинка с `title == artist` (Black Sabbath / Black Sabbath). |
| `R_self_aware` | **Самосознание** | `title` содержит `vinyl` / `винил` / `record` / `пластинка` / `analog`. |
| `R_meta_vertushka` | **Вертушка** | `title` или `artist` содержит `turntable` / `вертушк` / `gramophon` / `проигрыватель`. |
| `R_long_title` | **Поэма** | Пластинка с `title` длиннее 100 символов. |

### 5.4. Платформенные (отложены до публичного запуска)

Не выкатываем до момента, когда есть смысл говорить «первая сотня/тысяча». Помечаем как hidden в каталоге, но evaluator выключен флагом `RANDOM_PLATFORM_ENABLED=False` до запуска.

| Код | Название | Триггер |
|---|---|---|
| `R_first_thousand` | **Первая тысяча** | `User.id` в первой 1000 зарегистрированных. |
| `R_first_hundred` | **Первая сотня** | В первой 100. ⚫ Легенда. |
| `R_completionist` | **Завершитель** | Получил 30+ других ачивок. Мета-завершитель, тоже скрыт. |

### 5.5. Что выкинули

- **Зеркала** (`R_mirror`, `R_double_copy` — пластинка в коллекции и вишлисте / в двух коллекциях). Это поведенческий баг, а не пасхалка — не должно «награждаться».
- **Социальные синхронии** (`R_synchronicity`, `R_mentor`, `R_disciple`). Слишком сложная агрегация по платформе, отложено.

## 6. Награды

Только косметика. Никаких premium-фич, валют, шопа.

| Тип награды | Где применяется | Откуда выдаётся |
|---|---|---|
| **Эмалевый пин** | Раздел «Ачивки», превью на профилях (топ-3 свежих) | Каждая ачивка |
| **Рамка карточки** | Карточки записей в коллекции | Мета-ачивки серий (бронза → серебро → золото → платина → холо) |
| **Тема приложения** | Альтернативный визуал UI | META_geography → «Globus», META_eras → «Винтаж» |
| **Title** | Под ником в публичном и in-app профиле, в share-card | META_foundation, META_genres, META_evangelist, META_depth |
| **Иконка коллекции** | Кастомная иконка пользовательской коллекции | META_gifts → «Box», META_depth → «🎚 Мастер-копии» |
| **Анимированный пин** | 3D-вращение в разделе «Ачивки» | Только Легенда и мета-ачивки серий |

**Принцип:** одна Легенда = одна уникальная косметическая вещь. Косметика **не дублируется** между ачивками — каждая привязана к конкретной истории.

## 7. Технический каркас

### 7.1. Модель данных

```python
class UserAchievement(Base):
    __tablename__ = "user_achievements"
    id = Column(UUID, primary_key=True, default=uuid4)
    user_id = Column(UUID, ForeignKey("users.id", ondelete="CASCADE"), nullable=False, index=True)
    code = Column(String(64), nullable=False, index=True)  # 'C2_limited_x25' или 'H2:king-crimson'
    unlocked_at = Column(DateTime(timezone=True), nullable=False, default=now)
    progress = Column(Integer, default=0, nullable=False)
    progress_target = Column(Integer, default=0, nullable=False)
    is_unlocked = Column(Boolean, default=False, nullable=False, index=True)
    metadata = Column(JSONB, nullable=True)  # для H2/H5 — имя артиста/лейбла

    __table_args__ = (UniqueConstraint("user_id", "code"),)
```

### 7.2. Расположение кода

```
Backend/app/services/achievements/
├── registry.py              # каталог ачивок
├── evaluator.py             # ядро
├── share_card.py            # генерация PNG для шеринга
├── definitions/
│   ├── series/              # серии по файлам
│   │   ├── foundation.py
│   │   ├── scale.py
│   │   ├── rarity.py
│   │   ├── geography.py
│   │   ├── eras.py
│   │   ├── genres.py
│   │   ├── gifts.py
│   │   ├── community.py
│   │   ├── invitations.py
│   │   └── discography.py
│   └── random.py
└── tasks.py                 # фоновые (раз в сутки — H, INV_active_circle, R_thirty_three cooldown)
```

### 7.3. События

Эмитятся из API-обработчиков в той же транзакции, обрабатываются evaluator'ом синхронно. Дорогие проверки (H-категория) идут в `daily_tick`.

Ключевые события: `collection_item_added`, `wishlist_item_added`, `gift_booked`, `gift_completed`, `gift_received`, `follow_created`, `follow_received`, `profile_view` (только если viewer ≠ owner), `profile_shared_enabled`, `avatar_set`, `user_registered_via_ref`, `daily_tick`.

### 7.4. Идемпотентность

- `unlock_achievement` — upsert по `(user_id, code)`. Двойной event = один анлок.
- `update_progress` — пишем только если новое значение `>` текущего.
- Evaluator'ы — pure, без side-effects.

### 7.5. Анти-фарм

| Угроза | Защита |
|---|---|
| Накачал 1000 пластинок и удалил → B-серия | `COUNT(DISTINCT record_id)` от записей старше 24 часов. |
| Подарки циклом «друг ↔ друг» → J3/J4 | J3/J4 считают **разных получателей**. |
| Накачал followers ботами → K | Фолловер должен иметь ≥10 пластинок и аккаунт ≥30 дней. |
| Просмотры профиля сам себе → K5/K6 | View не инкрементится, если `viewer_user_id == owner_id`. |
| Подбор «ровно 33» → R_thirty_three | 24h cooldown без новых добавлений. |
| Накрутка приглашений → INV | Приглашённый должен иметь ≥10 пластинок и быть активным ≥30 дней. |

### 8.4. Шеринг в Instagram Stories

При тапе на конкретный пин в гриде / в анлок-анимации → кнопка «Поделиться».

**Генерация:** `POST /api/achievements/me/share-card/{code}` → возвращает URL PNG `1080×1920` (Stories 9:16).

**Композиция картинки:**
```
┌─────────────────────────┐  ← 1080×1920
│   градиентный фон       │
│   (цвет тира)           │
│                         │
│   [крупный пин 700×700] │  ← центр
│                         │
│   «Тёзка»               │  ← название, белый, 80px
│   Редкая · 10 мая 2026  │  ← тир + дата, 36px
│                         │
│   [аватар] @username    │  ← нижний блок, ~120 от низа
│   vinyl-vertushka.ru    │  ← логотип/ссылка, мелко
└─────────────────────────┘
```

**Автокроп под другие форматы:**
- 1080×1080 (Instagram Feed) — обрезаем 420 px сверху и снизу.
- 1080×1350 (Instagram Portrait) — обрезаем 285 px сверху и снизу.
- 1080×1920 (Telegram Stories, TikTok) — оригинал.

**Бэкенд:** Pillow + предзагруженные SVG-пины через `cairosvg`. Кэш по `code` (картинка не зависит от user-данных, кроме нижнего блока — отдельный композитинг username/avatar поверх).

**Mobile:** после генерации показываем `react-native-view-shot` overlay с превью + Share Sheet (iOS/Android share API → Instagram, Telegram, прочее).

### 8.5. Анимация анлока

Поверх экрана: затемнение → пин падает на проигрыватель → пин крутится 360° → разворачивается лента с названием + тиром. 2.2 сек до интерактивной фазы. Кнопки: «Поделиться» / «Закрыть».

**Правило батчинга:** в одной серии (`B*`, `J*`, …) при батч-анлоке показываем только самый высокий тир, остальные начисляем молча с пометкой «и ещё 3 на пути к META_scale».

### 8.6. Push

- Только в момент анлока, не чаще 1 push в час (батчинг).
- Рандомные ачивки — без push, только всплывающий тост в приложении («Ого, ты нашёл сюрприз»). Сохраняем эффект «не может быть».
