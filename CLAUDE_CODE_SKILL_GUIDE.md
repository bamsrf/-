# Как сделать свой первый скилл в Claude Code

Гайд для тех, кто никогда не программировал и боится кода.

---

## Что такое скилл?

**Скилл** — это инструкция для Claude, которую ты пишешь один раз, а потом вызываешь командой.

Пример: ты создал скилл `/review` — теперь можешь написать `/review` и Claude проверит твой код по твоим правилам.

**Зачем это нужно?**
- Не объяснять Claude одно и то же каждый раз
- Получать ответы в нужном тебе формате
- Автоматизировать рутину

---

## Что понадобится

- [ ] Claude Code (уже установлен, раз ты это читаешь)
- [ ] Текстовый редактор (VS Code, Cursor или даже Блокнот)
- [ ] 10 минут времени

---

## Шаг 1: Создай папку для скилла

Открой терминал и выполни команду:

```bash
mkdir -p ~/.claude/skills/review
```

**Что это делает:** создает папку `review` внутри `~/.claude/skills/`

**Где это находится:**
- На Mac: `/Users/твоеимя/.claude/skills/review/`
- На Windows: `C:\Users\твоеимя\.claude\skills\review\`

> Папка `.claude` скрытая (начинается с точки). Если не видишь её — включи показ скрытых файлов в настройках.

---

## Шаг 2: Создай файл SKILL.md

В папке `review` создай файл с именем `SKILL.md`

Полный путь: `~/.claude/skills/review/SKILL.md`

---

## Шаг 3: Напиши содержимое скилла

Открой `SKILL.md` и вставь это:

```yaml
---
name: review
description: Проверяет код на ошибки, безопасность и читаемость
---

# Инструкция для Code Review

Когда я прошу проверить код, сделай следующее:

## 1. Краткое резюме
Напиши 1-2 предложения: что делает этот код.

## 2. Проблемы (если есть)
Проверь на:
- Баги и логические ошибки
- Проблемы безопасности (SQL-инъекции, XSS и т.д.)
- Утечки памяти или ресурсов

Формат: таблица с колонками: Проблема | Где | Как исправить

## 3. Улучшения
Что можно сделать лучше:
- Читаемость кода
- Производительность
- Соответствие лучшим практикам

## 4. Оценка
Поставь оценку от 1 до 10 и объясни почему.

## Важно
- Не придумывай проблемы, которых нет
- Если код хороший — так и скажи
- Отвечай по-русски
```

---

## Шаг 4: Проверь, что скилл работает

Открой Claude Code и напиши:

```
/review
```

Claude должен ответить, что готов проверить код.

**Или покажи ему код:**

```
/review вот мой код:

function add(a, b) {
  return a + b;
}
```

---

## Как это работает: разбор файла

```yaml
---
name: review                          # имя команды (пишешь /review)
description: Проверяет код на ошибки  # когда Claude сам решает, применять ли скилл
---
```

Часть между `---` называется **frontmatter** — это настройки.

Всё что после — инструкции для Claude на обычном языке.

## Ещё один пример: скилл для коммитов

Создай папку и файл:

```bash
mkdir -p ~/.claude/skills/commit
```

Содержимое `~/.claude/skills/commit/SKILL.md`:

```yaml
---
name: commit
description: Создает git коммит с осмысленным сообщением
disable-model-invocation: true
---

# Инструкция для коммита

1. Посмотри какие файлы изменены (git status)
2. Посмотри что именно изменилось (git diff)
3. Напиши сообщение коммита по шаблону:

   тип(область): краткое описание

   Типы: feat, fix, refactor, docs, style, test

4. Выполни коммит

Пример сообщения: feat(auth): добавить проверку токена
```

Использование: `/commit`

---

## Настройки скилла (необязательные)

| Настройка | Что делает | Пример |
|-----------|------------|--------|
| `name` | Имя команды | `review` |
| `description` | Описание (когда применять) | `Проверяет код` |
| `disable-model-invocation: true` | Claude не будет сам вызывать скилл, только ты | Для опасных действий |
| `allowed-tools` | Какие инструменты может использовать | `Read, Grep` |

---

## Типовые ошибки и как починить

### 1. Скилл не появляется в списке

**Проблема:** Создал скилл, но `/review` не работает.

**Решение:**
- Проверь путь: файл должен быть `~/.claude/skills/review/SKILL.md`
- Не `SKILL.txt`, а именно `SKILL.md`
- Название папки = название команды
- Перезапусти Claude Code

---

### 2. Ошибка парсинга YAML

**Проблема:** Claude ругается на формат файла.

**Решение:**
- Проверь, что frontmatter начинается и заканчивается на `---`
- Нет лишних пробелов перед `---`
- После двоеточия всегда пробел: `name: review` (не `name:review`)

**Неправильно:**
```yaml
---
name:review
---
```

**Правильно:**
```yaml
---
name: review
---
```

---

### 3. Скилл срабатывает когда не надо

**Проблема:** Claude сам вызывает скилл, хотя ты не просил.

**Решение:** Добавь в frontmatter:
```yaml
disable-model-invocation: true
```

Теперь скилл работает только по команде `/review`.

---

### 4. Кириллица в названии

**Проблема:** Назвал скилл `/ревью` — не работает.

**Решение:** Название скилла только латиницей, цифрами и дефисом:
- `/review` — работает
- `/code-review` — работает
- `/ревью` — не работает
- `/review_code` — не работает (нельзя `_`)

---

### 5. Файл в неправильной папке

**Проблема:** Положил `SKILL.md` прямо в `~/.claude/skills/`

**Решение:** Скилл должен быть в своей папке:
```
~/.claude/skills/review/SKILL.md    # правильно
~/.claude/skills/SKILL.md           # неправильно
```

---

### 6. Слишком длинное описание

**Проблема:** Написал огромную инструкцию, Claude её игнорирует.

**Решение:**
- Описание в frontmatter — короткое (1-2 предложения)
- Подробные инструкции — в теле файла после `---`

---

### 7. Скилл для проекта, а не глобальный

**Проблема:** Хочешь скилл только для одного проекта.

**Решение:** Создай папку внутри проекта:
```
твой-проект/.claude/skills/review/SKILL.md
```

Этот скилл будет работать только в этом проекте.

---

## Где хранятся скиллы

| Тип | Путь | Когда работает |
|-----|------|----------------|
| Глобальный | `~/.claude/skills/имя/SKILL.md` | Везде |
| Проектный | `./проект/.claude/skills/имя/SKILL.md` | Только в этом проекте |

---

## Мини-шпаргалка: если ничего не работает

```
1. Проверь путь:     ~/.claude/skills/ИМЯ/SKILL.md
2. Проверь имя:      SKILL.md (не skill.md, не SKILL.txt)
3. Проверь YAML:     --- в начале и конце, пробел после :
4. Перезапусти:      Закрой и открой Claude Code
5. Проверь команду:  /имя (латиницей, без пробелов)
6. Покажи скиллы:    Напиши "какие скиллы доступны?"
7. Права доступа:    chmod 644 ~/.claude/skills/review/SKILL.md
8. Скрытые папки:    .claude начинается с точки — включи показ
9. Логи:             Смотри что пишет Claude при запуске
10. Пересоздай:      Удали папку скилла и создай заново
```

## Готово!

Ты создал свой первый скилл. Теперь можешь:
- Создать скилл для любой повторяющейся задачи
- Настроить Claude под свой стиль работы
- Делиться скиллами с командой (просто скопируй папку)

**Следующий шаг:** попробуй изменить инструкции в скилле `/review` под себя — добавь свои правила проверки кода.
